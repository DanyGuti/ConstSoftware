Propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH y DELETE

POST

El método de POST se usa más para crear nuevos recursos. Cuando se realiza un buen proceso de posteo se obtiene un HTTP code de 201.

GET

El método de GET se usa más para hacer retrieve de información o realizar lectura de algún recurso. Cuando se realiza un buen proceso de posteo se obtiene un HTTP code de 201. No puede cambiar información solamente leerla. Es un método idempotente, pues siempre al realizar múltiples requests, únicamente se realizará una.

PUT

El método de PUT se usa para actualizar capacidades.

HEAD

El método de HEAD pide respuesta idéntica a la de una petición GET, pero sin el cuerpo de la respuesta..

PATCH

El método de PATCH utilizado para aplicar modificaciones parciales a un recurso.

DELETE

El método de DELETE utilizado para borrar un recurso específico.
